fix: Комплексное исправление Frontend + Backend интеграции

## Проблема
API возвращает данные в формате `{success: true, data: [...], timestamp: ...}`
Но Frontend ожидал данные напрямую как массив/объект.

Результат: `openPositions.slice is not a function` и белый экран.

## Исправления

### Frontend - Все страницы (10 файлов)
Исправлено извлечение данных из API response:

1. Dashboard.tsx - добавлено извлечение .data для summary, metrics, chartData, positions, signals
2. Positions.tsx - positions?.data || []
3. Pairs.tsx - pairs?.data || []
4. Analytics.tsx - все 5 data hooks
5. Backtest.tsx - results?.data || []
6. Settings.tsx - settings?.data и presets?.data
7. Alerts.tsx - alertSettings?.data и alertHistory?.data
8. System.tsx - systemStatus?.data и logs?.data

### Backend - WebSocket endpoint
- websocket.py - полноценный WebSocket на /ws
- Heartbeat каждые 30 сек
- Каналы: signals, positions, system
- Подписка/отписка

### Backend - Модели и Routes
- models.py - все Pydantic модели
- pairs.py - реальные данные из БД
- analytics.py - реальные данные из БД
- settings.py - исправлен синтаксис
- main.py - подключен websocket router
